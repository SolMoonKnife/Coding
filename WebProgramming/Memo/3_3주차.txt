요즘은 시맨틱 웹(문서 탐색이 용이하게 구조화된 웹)대신
원 페이지 웹(복잡한 자바 스크립트로 동작하는 웹)으로 개발되는 추세.

시맨틱 태그 위치, 색, 모양이 자동으로 결정되지 않음(기본적으로 div태그와 동일)
-> CSS3를 이용해 직접 위치, 색, 모양을 지정하자

사이즈를 수치로 입력하면 픽셀, %로 입력하면 프레임 최대치 기준 비율

[폼 태그]: 폼을 만드는 다양한 태그. 사용자가 특정한 정보를 입력할 수 있는
인터페이스를 폼이라고 한다. 예를 들어서 아이디, 패스워드, 로그인 버튼이
하나의 폼으로 묶인다.

화면이 새로고침되지 않는다는 것->서버에 새로운 페이지를 요청하지 않는다.
로그인 시도 시 패스워드를 입력하지 않고 로그인 버튼을 누름->
잘못된 동작임을 브라우저 단계에서 인식하고 처리할 수 있다.
ㄴ자바 스크립트의 클라이언트 스크립트 활용

액션: 
폼은 서버로 입력 데이터를 보내기 위한 것이다.

https://search.naver.com/search.naver?where=nexearch&sm=top_hty&fbm=0&ie=utf8&query=bts

https:// <-[프로토콜]
search.naver.com <-[URL]
search.naver <-[서버 프로그램]
? <-[매개 변수]

텍스트 박스, 체크 박스, 라디오 박스, 스핀 버튼 등은 모두 input임
type를 명시해 구분해야 함

enctype="데이터 인코딩 타입" <!--파일 입력 시 필요-->

한 그룹에 체크/라디오박스는 name속성 값이 같아야 함
화면에 보이는 것과 value속성값은 다를 수 있는데, 서버로 전달되는 것은 value값
